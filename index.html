While checking the latest features, I also noticed that an older feature was finally also supported by Chrome late 2023, and Safari not too long before that. One of the challenges I might face from now and then is consistency in content that should be consistent and similar. Examples could be a blog or page of some sort that shows articles, recipes or similar that dont need to have very different layout, and rather often can go out of hand on nesting grids.

Subgrid is a grid-template option that inherits the cols and rows from its parent. Usually when creating a grid, only the direct children gets placed within a grid, and next line would follow a flow which often might get messy. Instead of nesting grids to match, use subgrid!

I would not use subgrid for everything, but in cases where we have uniform layouts for a section, it could be advantageous to maybe even sit back and consider to increase the complexity of the main grid structure of the page, as this can help give more freedom to aligning objects/elements as you would like them

Despite being soon outdated and the material page linked to blog posts suggesting not to use it, I decided to go through the NetNinja tutorial to explore what SASS is all about. After the first three lessons, I became frustrated with a deprecation warning in the terminal while running Gulp. This warning highlighted the transition from the initial gulpfile.js to gulpfile.mjs, which required updating the import syntax for Dart Sass to import * as sass from 'sass' and switching to a modern module format.

This change provided better compatibility with ECMAScript modules and also allowed for the direct compilation of SCSS files without relying on the deprecated JS API. Additionally, I updated the Sass syntax to utilize @use instead of @import, in some locations. This transition required that all references to variables be prefixed with the module name (e.g., variables.$primary), which enhances code organization and helps avoid naming conflicts.

To further improve the Gulp setup, the tutorial incorporated the gulp-purgecss plugin to remove unused CSS from the final output (which was a relief) based on the HTML files present in my project. This addition helps optimize the size of the generated CSS file, ensuring that only the styles in use are included. As I continue to work through the tutorial, I am gaining a deeper understanding of Sass, its features, and how to manage CSS effectively in modern web development.

Transitions and animations in CSS add dynamic, interactive elements to web design by smoothly changing properties like color, size, or position over a specified duration.

While transitions respond to user interactions (like hovering or clicking), animations allow continuous, complex movements, making designs more engaging and visually appealing. SVG Illustrations

Creating SVGs in Inkscape and testing them in Notepad++ or Visual Studio Code requires time - and creativity.

Going through tutorials and mostly spending time testing to create figures that are symmetrical and in the way I envisioned them, taught me how important it is to maintain control over every object and path. Hidden or unused symbols can clutter the file and cause unexpected issues. By cleaning the SVG structure and testing changes directly in the code, I ensure that every detail—whether visible or not—behaves as intended. It’s a meticulous process, but it makes animations smoother and the final output much more reliable.
